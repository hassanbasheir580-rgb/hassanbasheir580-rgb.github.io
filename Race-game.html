Client-Server Multiplayer Race Game

The Client-Server Multiplayer Race Game is an Operating Systems project developed in C that implements a multiplayer game using a client-server architecture. The application uses TCP socket programming to allow multiple clients to connect to a central server, enabling real-time communication and interactive gameplay. The server manages client connections, processes game events, and maintains the shared game state throughout each session.

The project demonstrates key operating systems and networking concepts, including socket programming, client-server communication, and synchronization between multiple connected clients. The codebase is organized into separate modules for networking, game logic, and shared components, making the application easier to maintain and extend while following good software engineering practices.

Developing this project strengthened my understanding of systems programming and network communication in C. It provided hands-on experience with designing reliable client-server applications, debugging network interactions, and applying theoretical operating systems concepts to a practical, real-world software project.
